3. EDRMS Strategy Paper

Transport for NSW

Presented July 2018 EDRMS Hosted Azure IaaS

The enterprise vision was to treat Data and Content as a shared Enterprise Asset which outlined the need to connect technology layers with the broader Ecosystem requiring a holistic approach across multiple agency's to EEDRMS capability. Review of the current EDRMS system within the above context (Agency specific), identified a number of options to introduce this capability taking into account current enterprise infrastructure and future target state architecture. From a business perspective moving to an enterprise wide EDRMS facilitates consolidation of existing TfNSW document and record management processes and digital asset storage to achieve an enterprise wide EDRMS. The concepts presented covered current and long term implementation approach using API & Integration Technologies and consolidation of various EDRMS systems with redefined Information Architecture Model.

4. Route Planning Service

Transport Networks

Deployed April 2018 Serv Model SaaS Azure

A Transport agency trialed with an external vendor to utilise a cloud based SaaS digital route planning design tool. A 12 month pilot subscription was obtained to set up transport cluster network accounts in order to complete the evaluation. The pilot addresses the short term transport scenario planning needs for a number of high profile projects. Assessment of the service was completed to determine fit for purpose scenario route planning for multi modal planning capability across the transport cluster. An assessment framework and functional criteria was defined to facilitate fit-for-purpose digital route planning application integration. An review of internal planning systems was completed to compare digital map draw capability in the event that this capability is provisioned as part of the enterprise wide strategic solution. Outcome from the review provided recommendation for short term and longer term route planning capability in line with enterprise wide network planning strategy across different modes of transport using SaaS hosted on Azure.

5. Application Migration

Timetable Planning

Deployed Dec 2017 Model PaaS AWS

As part of the Customer Channel Transformation Program (CCTP) initiative, migration of Timetable planning service was implemented from externally hosted physical infrastructure to a cloud hosted (AWS) solution. The overall objective was to align with future strategic business objectives and minimise any business impact or risk as a result of Application migration and provisioning the planning service as a self managed Service through Software as a Service. This allowed Transport to deliver more effective Transport timetable data to downstream applications (i.e. utilising API end points) and on-demand access to secondary dependant environments (ie SIT, UAT) for data preparation and deployment. Improved efficiences were obtained as a result of groups application / data producers and consumers within the same operating(cloud)domain. Key elements used within this assignment were; SOA Patterns, responsive and adaptive web technologies.
